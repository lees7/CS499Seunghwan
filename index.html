Program Outcomes & Core Competencies

1. Software Design and Engineering

A professional software engineer must look beyond basic functionality and focus on modularity, maintainability, and architectural clean code. I demonstrated this competency by enhancing a 3D Computational Graphics project originally developed in CS-330. I restructured a procedural, highly coupled codebase into an organized, reusable component-based architecture using C++, OpenGL, GLFW, GLEW, and GLM. I integrated advanced graphics pipelines, including the Phong lighting model, interactive camera controls, depth testing, and dynamic light animations. This enhancement proves my ability to read legacy code, integrate industry-standard third-party libraries, and refactor code into clean structures.

2. Algorithms and Data Structures

Selecting the right data structure is fundamental to application scalability and performance. I demonstrated my ability to analyze and optimize software performance by refactoring a backend service system originally developed in CS-320. The original project relied on unoptimized data handling. I redesigned the backend components (ContactService, TaskService, and AppointmentService) by implementing HashMap-based storage in Java. This structural shift optimized record insertion, lookup, and deletion down to average constant time complexity, O(1). Furthermore, I designed search, filtering, and sorting algorithms to enable advanced keywords and description-based queries.

3. Databases

Modern software systems rely heavily on robust data layer abstraction to keep applications scalable and secure. I demonstrated my database design capabilities by enhancing a Client/Server dashboard application originally developed in CS-340 for Grazioso Salvare. I decoupled the user interface from the data layer by developing a fully reusable, modular Python CRUD module for MongoDB. This tier separation isolated the database query logic from the presentation dashboard. I implemented full Create, Read, Update, and Delete functions, completed tight database connection validations, and built dynamic queries to feed real-time visual data directly into interactive dashboard components.

4. Software Testing and Quality Assurance

Writing functional code is only half the battle; ensuring it remains reliable through iterative updates is what separates amateur code from professional software. During my data structures enhancement (CS-320 Java backend), I explicitly expanded the JUnit testing framework. I wrote comprehensive automated unit tests covering all new searching, filtering, and sorting algorithms. This process validated my algorithm correctness and ensured zero regression on existing code, demonstrating my deep commitment to software quality assurance and test-driven mindsets.
