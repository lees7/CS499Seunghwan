Enhancement Two: Algorithms and Data Structures

Artifact Overview

The artifact selected for this enhancement was originally developed in CS-320 Software Test, Automation, and Quality Assurance. The project consisted of three backend service classes: ContactService, TaskService, and AppointmentService. These services were designed to manage contacts, tasks, and appointments while enforcing validation rules and maintaining data integrity.

The original implementation provided basic CRUD functionality, including creating, updating, retrieving, and deleting records. While functional, the project offered opportunities for enhancement through improved data structures, search capabilities, filtering functionality, sorting algorithms, and expanded testing.

Enhancements Implemented

  • Optimized backend storage using HashMap data structures
  • Implemented O(1) record retrieval by unique identifier
  • Added contact search by first name, last name, and phone number
  • Added task search by task name and description keywords
  • Added appointment search by description keyword
  • Implemented filtering by area code for contacts
  • Implemented filtering by date range for appointments
  • Implemented task filtering by name prefix
  • Added sorting algorithms for contacts, tasks, and appointments
  • Expanded JUnit testing for new functionality

Algorithms and Data Structure Improvements

A major focus of this enhancement was improving the efficiency and scalability of the backend services. The system utilizes HashMap data structures to store contacts, tasks, and appointments, allowing insertion, deletion, and retrieval operations to execute in average constant time complexity, O(1). This design choice significantly improves performance as the number of stored records increases.

Additional algorithmic functionality was implemented to improve data organization and retrieval. Search algorithms allow users to locate records using names, phone numbers, descriptions, and other attributes. Filtering operations return subsets of records matching specific criteria, while sorting algorithms organize data into meaningful order for improved usability.

Algorithm Complexity Analysis:
  • HashMap Insert Operations: O(1)
  • HashMap Retrieval Operations: O(1)
  • HashMap Delete Operations: O(1)
  • Search Operations: O(n)
  • Filtering Operations: O(n)
  • Sorting Operations: O(n log n)

Testing and Validation

To ensure the correctness of the enhanced algorithms and data structures, the project's JUnit test suites were expanded to include validation of search, filtering, sorting, retrieval, update, and deletion functionality. Additional test cases were developed to verify both successful operations and error-handling behavior.

Comprehensive testing helped ensure that all enhancements operated correctly while maintaining system reliability and data integrity. The testing process also strengthened the maintainability of the project and reduced the likelihood of future defects.

Reflection

Through this enhancement process, I gained a deeper understanding of how data structures and algorithms directly influence software performance and scalability. Implementing HashMaps reinforced the importance of selecting the appropriate data structure based on application requirements and performance goals.

Developing search, filtering, and sorting functionality provided valuable experience in algorithm design and complexity analysis. One of the most important lessons learned was how small design decisions can significantly impact system efficiency and user experience as applications grow in size and complexity.

A significant challenge involved integrating new algorithms into the existing architecture while maintaining reliability and readability. Expanding automated testing and validating each enhancement helped ensure that the project remained stable and maintainable throughout the development process.
