Enhancement Two: Algorithms and Data Structures

Artifact Overview

The artifact selected for this enhancement was originally developed in CS-320 Software Test, Automation, and Quality Assurance. The project consists of ContactService, TaskService, and AppointmentService classes that provide backend functionality for managing contacts, tasks, and appointments. The original implementation focused on CRUD operations, validation, and software testing to ensure data integrity and reliability.

While the original artifact successfully managed application data, it provided opportunities to improve efficiency, scalability, and data retrieval capabilities through more advanced data structures and algorithms. For this enhancement, I redesigned portions of the backend services to incorporate optimized storage, searching, filtering, and sorting operations.

Enhancements Implemented

  • Implemented HashMap-based storage for efficient data management
  • Added contact search by first name, last name, and phone number
  • Added task search by task name and description keyword
  • Added appointment search by description keyword
  • Implemented filtering operations for contacts, tasks, and appointments
  • Added sorting functionality for contacts, tasks, and appointments
  • Expanded backend retrieval methods
  • Added task counting and collection management functionality
  • Expanded JUnit testing to validate all enhancements

Algorithms and Data Structure Improvements

A primary goal of this enhancement was improving performance and scalability. HashMap data structures were utilized throughout the service classes to provide efficient record storage and retrieval. Using HashMaps allows insertion, lookup, and deletion operations to execute in average constant time complexity, O(1), making the services significantly more efficient as the number of stored records increases.

Additional search algorithms were implemented to allow users to locate records using names, phone numbers, descriptions, and keywords. Filtering functionality was added to return records matching specific criteria, while sorting operations were implemented to organize information into meaningful order for improved usability and accessibility.

Algorithm Complexity Analysis:
  • HashMap Insert Operations: O(1)
  • HashMap Retrieval Operations: O(1)
  • HashMap Delete Operations: O(1)
  • Search Operations: O(n)
  • Filtering Operations: O(n)
  • Sorting Operations: O(n log n)

Testing and Validation

To verify the correctness of the enhanced functionality, I expanded the existing JUnit test suites to include search, filtering, sorting, retrieval, update, and deletion operations. These tests helped validate algorithm correctness and ensured that the enhancements did not negatively impact existing functionality.

The testing process strengthened overall software reliability and reinforced the importance of automated testing when implementing new features within an existing codebase.

This enhancement strengthened my understanding of how data structures and algorithms directly impact software performance. I also gained practical experience developing search, filtering, and sorting functionality within a real-world backend application.

One challenge involved integrating new algorithmic functionality while maintaining readability and reliability throughout the existing system. Expanding the JUnit test suite helped verify correctness and reinforced the importance of testing in professional software development.

Overall, this enhancement improved my skills in algorithm optimization, data structure selection, backend service development, software testing, and performance analysis. The completed artifact demonstrates my ability to evaluate existing software, identify improvement opportunities, and implement efficient solutions that provide additional value to users.

Enhancement Three: Databases

Artifact Overview

The artifact selected for this enhancement was originally developed in CS-340 Client/Server Development. The project involved creating a database-driven dashboard application for Grazioso Salvare, an animal rescue organization that uses animal shelter data to identify candidates for specialized rescue operations. The original application connected to a MongoDB database and displayed animal records through a web-based dashboard interface.

While the original project successfully demonstrated database connectivity and data visualization, opportunities existed to improve maintainability, modularity, and database interaction. For this enhancement, I redesigned the database layer by implementing a reusable CRUD module and improving the overall architecture of the application. These changes created a more scalable solution while demonstrating advanced database management skills.

Enhancements Implemented

  • Developed a reusable CRUD Python module for MongoDB operations
  • Implemented Create, Read, Update, and Delete database functionality
  • Added database connection validation and error handling
  • Separated database logic from dashboard presentation logic
  • Improved MongoDB query organization and maintainability
  • Implemented dynamic filtering for rescue categories
  • Integrated database queries with interactive dashboard components
  • Improved data retrieval and visualization workflows
  • Enhanced application scalability through modular design

Database Improvements

The primary focus of this enhancement was improving database design and software architecture. The original project accessed MongoDB directly within the dashboard application, which tightly coupled the user interface with database operations. To improve maintainability, I developed a dedicated CRUD module that encapsulates all database functionality into reusable methods.

The CRUD module provides separate methods for creating, reading, updating, and deleting records within the MongoDB collection. This approach follows software engineering best practices by separating concerns and reducing dependencies between application components. As a result, future modifications to database operations can be made without significantly impacting the dashboard interface.

Additional improvements included implementing more efficient MongoDB queries to support Grazioso Salvare's rescue requirements. The dashboard dynamically filters records based on breed, age, and sex characteristics associated with Water Rescue, Mountain/Wilderness Rescue, and Disaster/Individual Tracking operations. These queries provide users with meaningful data subsets while maintaining responsiveness and usability.

Testing and Validation

Extensive testing was performed to validate database connectivity, CRUD functionality, and dashboard integration. Individual CRUD methods were tested to verify successful record insertion, retrieval, updating, and deletion. Additional testing confirmed that dashboard filters correctly returned matching records from MongoDB and updated associated visualizations.

Error handling was also tested to ensure the application responded appropriately when invalid data or connection issues occurred. These validation procedures improved overall reliability and demonstrated the importance of robust database testing practices.

Reflection

This enhancement strengthened my understanding of database management, client-server architecture, and data-driven application development. Designing a reusable CRUD module demonstrated the value of modular programming and reinforced the importance of separating database logic from presentation logic.

One challenge involved designing MongoDB queries that efficiently supported the rescue operation requirements while maintaining dashboard performance. Addressing this challenge improved my ability to work with NoSQL databases and optimize database interactions within larger software systems.

Overall, this enhancement improved my skills in database design, MongoDB administration, Python application development, software architecture, and data visualization integration. The completed artifact demonstrates my ability to design and implement database solutions that support real-world business requirements while maintaining scalability, reliability, and maintainability.
